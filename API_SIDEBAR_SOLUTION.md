# API Sidebar Solution - Tag-Based Collapsible Groups

## Problem
The API documentation sidebar was showing as a flat list of 73+ endpoints instead of collapsible groups organized by OpenAPI tags.

## Root Cause
The `docusaurus-plugin-openapi-docs` plugin's `categoryLinkSource: "tag"` option generates `.tag.mdx` files that use `useCurrentSidebarCategory()`, which requires them to be actual Docusaurus category index pages. However, with autogenerated sidebars, these files are not properly associated with categories, causing build crashes.

## Solution
**Manually define the sidebar structure** in `sidebars.ts` using the actual OpenAPI tag organization from `sources/swagger.json`.

### Configuration Changes

#### 1. docusaurus.config.ts
```typescript
config: {
  emailengine: {
    specPath: "sources/swagger.json",
    outputDir: "docs/api",
    sidebarOptions: {
      groupPathsBy: "tag",           // Group by tags
      sidebarCollapsed: true,        // Make groups collapsible
      sidebarCollapsible: true,      // Enable collapse functionality
      // DO NOT USE: categoryLinkSource: "tag" - causes crashes
    },
    version: "2.57.0",
    label: "v2.57.0",
    baseUrl: "/docs/api",
    versions: {},
  }
}
```

#### 2. sidebars.ts
Manually define the sidebar structure with categories matching OpenAPI tags:

```typescript
apiSidebar: [
  {
    type: 'doc',
    id: 'api/emailengine-api',
    label: 'API Overview',
  },
  {
    type: 'category',
    label: 'Account',
    collapsed: true,
    items: [
      'api/get-v-1-accounts',
      'api/get-v-1-changes',
      // ... more endpoints
    ],
  },
  {
    type: 'category',
    label: 'Mailbox',
    collapsed: true,
    items: [
      'api/get-v-1-account-account-mailboxes',
      // ... more endpoints
    ],
  },
  // ... more categories
]
```

### File Naming Pattern
The plugin generates files with kebab-case naming:
- Operation ID: `getV1AccountAccount` тЖТ File: `get-v-1-account-account.api.mdx`
- Operation ID: `postV1OAuth2` тЖТ File: `post-v-1-oauth-2.api.mdx`

Note the important transformations:
- `v1` becomes `v-1`
- `oauth2` becomes `oauth-2`

## Maintenance

### Updating When OpenAPI Spec Changes

When you update `sources/swagger.json`, regenerate the API docs and sidebar:

```bash
# 1. Regenerate API documentation files
npm run docusaurus gen-api-docs all

# 2. Generate updated sidebar structure
npm run generate-api-sidebar

# 3. Copy the output from the script into sidebars.ts
#    Replace the apiSidebar array definition
```

The `generate-api-sidebar` script (`scripts/generate-api-sidebar.js`) automatically:
- Reads the OpenAPI spec
- Groups endpoints by tags
- Generates TypeScript code for sidebars.ts
- Shows a summary of all categories and endpoint counts

## Why This Approach Works

1. **Avoids plugin limitations** - No `.tag.mdx` files that require special category association
2. **Full control** - Complete control over sidebar structure and organization
3. **Type-safe** - TypeScript catches invalid document IDs at build time
4. **Maintainable** - Helper script regenerates structure when spec changes
5. **Clean URLs** - No extra category pages, direct links to endpoints

## Alternative Approaches That Don't Work

1. **`categoryLinkSource: "tag"`** - Creates `.tag.mdx` files that crash with autogenerated sidebars
2. **Folder-based organization** - Would require reorganizing 73+ generated files into folders
3. **Custom plugin configuration** - Plugin doesn't support dynamic category creation with autogenerated sidebars
